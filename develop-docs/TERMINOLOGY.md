# Terminology

Glossary of terms used in the Sentry Cocoa SDK, with a focus on main-thread responsiveness.

## Apple Definitions

| Term                     | Definition                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| ------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Hang**                 | The main run loop is unresponsive for [250 ms](https://developer.apple.com/documentation/xcode/understanding-hangs-in-your-app#Understand-hangs:~:text=250%20ms) or more. Measured as the time between [CFRunLoopActivity](https://developer.apple.com/documentation/corefoundation/cfrunloopactivity) states `.afterWaiting` and `.beforeWaiting`.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| **Hitch**                | A [frame that appears on screen later than expected](https://developer.apple.com/documentation/xcode/understanding-hitches-in-your-app) during a continuous interaction (scrolling, dragging, animation). Caused by a delay in the [commit or render phase](https://developer.apple.com/videos/play/tech-talks/10855/?time=435) of the [render loop](https://developer.apple.com/videos/play/tech-talks/10855/?time=128). The [hitch duration](https://developer.apple.com/documentation/xcode/understanding-hitches-in-your-app#Understand-frame-lifetime-and-hitch-duration) is only the overshoot past the frame deadline, not the total frame time.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| **Commit hitch**         | A hitch caused by the app's process [missing the commit deadline](https://developer.apple.com/documentation/xcode/understanding-hitches-in-your-app#Understand-commit-hitches-and-render-hitches) (~[8.3 to 16.7 ms](https://developer.apple.com/videos/play/tech-talks/10855/?time=443) depending on refresh rate), preventing the Core Animation commit from reaching the render server in time. Commit hitches and hangs [share the same root cause](https://developer.apple.com/documentation/xcode/understanding-hitches-in-your-app#Overview) — an unresponsive main thread — but differ by interaction type: during continuous interactions (scrolling, animation) it's a commit hitch; during discrete interactions (tap, press) it's a hang. This is the only subtype of hitch observable by the SDK.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| **Render hitch**         | A hitch caused by the [render server](https://developer.apple.com/documentation/xcode/understanding-hitches-in-your-app#Understand-commit-hitches-and-render-hitches) not finishing its work in time for the frame's [presentation time](https://developer.apple.com/videos/play/tech-talks/10855/?time=128). The app's commit finishes on time, but the render server's CPU or GPU work [misses its deadline](https://developer.apple.com/videos/play/tech-talks/10855/?time=502). Although render hitches happen out of process, they are [usually due to the app](https://developer.apple.com/documentation/xcode/understanding-hitches-in-your-app#Understand-commit-hitches-and-render-hitches) submitting overly complex rendering work. Cannot be observed directly by the SDK.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| **Watchdog termination** | The OS [terminates an app](https://developer.apple.com/documentation/xcode/addressing-watchdog-terminations) that [blocks the main thread for a significant time](https://developer.apple.com/documentation/xcode/addressing-watchdog-terminations#Overview), using termination code [`0x8badf00d`](https://developer.apple.com/documentation/xcode/addressing-watchdog-terminations#Overview). Covers both launch ([`scene-create`](https://developer.apple.com/documentation/xcode/addressing-watchdog-terminations#Interpret-the-app-responsiveness-watchdog-information)) and ongoing responsiveness ([`scene-update`](https://developer.apple.com/documentation/xcode/addressing-watchdog-terminations#Interpret-the-app-responsiveness-watchdog-information)) failures. The exact timeout is not specified by Apple and varies by context ([Apple's example](https://developer.apple.com/documentation/xcode/addressing-watchdog-terminations#Interpret-the-app-responsiveness-watchdog-information) shows ~19.97s for iOS foreground, [15s for watchOS](https://developer.apple.com/documentation/xcode/addressing-watchdog-terminations#Interpret-background-task-watchdog-information-watchOS) background tasks). Listed as [App Watchdog](https://developer.apple.com/documentation/xcode/reduce-terminations-in-your-app#Understand-termination-reasons) in Xcode Organizer termination metrics. |

## Sentry Definitions

These are the terms used in the Sentry SDK and product. They are inspired by Apple definitions but adapted for Sentry error reporting.

### Hang Detection

| Term              | Definition                                                                                                                                                                                                                                                                                                                                                   |
| ----------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Hang**          | The main run loop is unresponsive, detected by measuring the time the main run loop is blocked between `.afterWaiting` and `.beforeWaiting`. The duration is the total blocked time, including the per-frame budget. A 500 ms hang means the runloop was blocked for 500 ms. Reported as an issue with a sampled profile of the main thread during the hang. |
| **Major hang**    | A hang lasting more than 250 ms.                                                                                                                                                                                                                                                                                                                             |
| **Severe hang**   | A hang lasting more than 500 ms.                                                                                                                                                                                                                                                                                                                             |
| **Critical hang** | A hang lasting more than 1 second.                                                                                                                                                                                                                                                                                                                           |
| **Fatal hang**    | A hang during which the app exits. Reported as a fatal exception rather than a handled one.                                                                                                                                                                                                                                                                  |

### Hitch Detection

| Term            | Definition                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| --------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Hitch**       | A main run loop iteration that exceeds the frame deadline (~16 ms at 60 fps), measured only during interactive mode (touch input, scrolling). Unlike hangs, the duration is only the overshoot past the per-frame budget, not the total iteration time. A 20 ms runloop iteration at 60 fps is a 3.3 ms hitch. Hitches that cross the 250 ms threshold are tracked as hangs instead. Severity tiers based on overshoot: >10 ms, 5-10 ms, <5 ms. |
| **Hitch ratio** | The proportion of time, measured in milliseconds per second, that the app was showing a delayed frame during user interaction. Used to understand the severity of UI disruption across a session.                                                                                                                                                                                                                                               |
