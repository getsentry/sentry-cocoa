# Terminology

Glossary of terms used in the Sentry Cocoa SDK, with a focus on main-thread responsiveness and hang detection.

## Apple Definitions

| Term                     | Definition                                                                                                                                                                                                                                           | Source                                                                                                                      |
| ------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------- |
| **Hang**                 | The main thread is unresponsive for 250 ms or more. Measured as the time between CFRunLoop states `.afterWaiting` and `.beforeWaiting`.                                                                                                              | [Apple: Understanding hangs in your app](https://developer.apple.com/documentation/xcode/understanding-hangs-in-your-app)   |
| **Severe hang**          | A hang lasting longer than the watchdog timeout, typically around 20 seconds on iOS. The OS terminates the app.                                                                                                                                      | [Apple: Addressing watchdog terminations](https://developer.apple.com/documentation/xcode/addressing-watchdog-terminations) |
| **Hitch**                | A frame that misses its VSYNC display deadline in the render pipeline. Measured as time past the expected frame presentation. This is not a runloop concept. Hitches are about the render pipeline, covering the commit, render, and display phases. | [Apple: Explore UI animation hitches and the render loop](https://developer.apple.com/videos/play/tech-talks/10855/)        |
| **Commit hitch**         | A hitch caused by the main thread missing the commit deadline of around 8 to 16 ms depending on refresh rate. This is one subtype of hitch.                                                                                                          | [Apple WWDC 2021: Demystify and eliminate hitches](https://developer.apple.com/videos/play/wwdc2021/10252/)                 |
| **Watchdog termination** | The OS kills an app that has been unresponsive for too long, typically around 20 seconds. Detected post-mortem by comparing app state across launches.                                                                                               | [Apple: Addressing watchdog terminations](https://developer.apple.com/documentation/xcode/addressing-watchdog-terminations) |

## SDK Definitions

| Term                                 | Definition                                                                                                                                                                                                                                                    | Class / File                                     |
| ------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------ |
| **Runloop delay**                    | Any period where the main runloop is blocked. This is the time between `.afterWaiting` and `.beforeWaiting` that exceeds the polling interval. It is the lowest-level observation in the SDK.                                                                 | `RunLoopDelayTracker`                            |
| **Polling interval**                 | The sampling rate at which the `RunLoopDelayTracker` checks for runloop delays. Currently around 25 ms, which is 1.5 times the expected frame duration at 60 fps. This is not a hang threshold. It controls how frequently we detect that a delay is ongoing. | `DefaultRunLoopDelayTracker.pollingInterval`     |
| **App hang**                         | A runloop delay that exceeds a configured threshold. For example, 250 ms for hang events or 2 seconds for watchdog terminations. Classified by the `AppHangTracker` middle layer, not by the raw runloop observer.                                            | `AppHangTracker`                                 |
| **App hang threshold**               | The per-observer duration above which a runloop delay is classified as a hang. Different consumers can register different thresholds.                                                                                                                         | `AppHangTracker.addObserver(threshold:handler:)` |
| **ANR (Application Not Responding)** | Legacy term used in the older `SentryANRTracker`. Equivalent to an app hang. The V2 path using `enableWatchdogTerminationsV2` uses `AppHangTracker` instead.                                                                                                  | `SentryANRTracker`                               |
